2. Flujo de Precios — Cálculo Automático

flowchart LR
    A["Precio Base SKU\n(ej: S/ 5.00)"] --> D
    B["Tarifa Ruta por Ubigeo\n(ej: S/ 0.80)"] --> D
    D["PRECIO UNITARIO\n= Base + Tarifa Ruta"]
    D --> E["× Cantidad"]
    E --> F["TOTAL LÍNEA"]
    F --> G["Σ Líneas = TOTAL PEDIDO"]

    style D fill:#4338ca,stroke:#818cf8,color:#fff
    style G fill:#15803d,stroke:#4ade80,color:#fff
Fórmula: precioUnitario = precioBaseSKU + tarifaRuta

Ejemplo:
Papa Amarilla 1kg Ziplock = S/ 5.00 (base)
Ruta Norte SJL = S/ 0.80 (flete)

Precio = 5.00 + 0.80 = S/ 5.80 por kg

Rendimiento % = (Producto Terminado / Materia Prima) × 100
Merma = Materia Prima - Producto Terminado
Costo Real/kg = Costo Total MP / kg de Producto Terminado

5. Ciclo de Vida del Pedido

stateDiagram-v2
    [*] --> REGISTERED: Vendedor registra pedido
    REGISTERED --> DISPATCHED: Despachador confirma despacho
    REGISTERED --> CANCELLED: Vendedor cancela (con motivo)
    DISPATCHED --> ON_ROUTE: Chofer sale a repartir
    ON_ROUTE --> DELIVERED: Chofer confirma entrega
    ON_ROUTE --> ISSUE: Chofer reporta novedad
    DELIVERED --> COLLECTED: Chofer registra cobro
    ISSUE --> REGISTERED: Se reprograma
    COLLECTED --> [*]
    CANCELLED --> [*]

    note right of REGISTERED
        Precio calculado automáticamente
        Stock PT aún no se descuenta
    end note

    note right of DISPATCHED
        Stock PT se descuenta
        Se genera hoja de ruta
    end note

    note right of COLLECTED
        Se registra:
        - Monto cobrado
        - Método de pago
        - N° comprobante
    end note

8. Flujo Técnico — Request Lifecycle

sequenceDiagram
    participant U as Usuario (Browser)
    participant N as Next.js (Frontend)
    participant A as NestJS (API)
    participant G as Auth Guard
    participant R as RBAC Guard
    participant S as Service
    participant P as Prisma
    participant DB as PostgreSQL

    U->>N: Click "Nueva Venta"
    N->>A: POST /api/orders (JWT en header)
    A->>G: Valida JWT
    G-->>A: ✓ Token válido, userId extraído
    A->>R: Verifica permiso "orders.create"
    R->>P: Busca roles y permisos del user
    P->>DB: SELECT roles, permissions
    DB-->>P: Roles + permisos
    P-->>R: Permisos efectivos
    R-->>A: ✓ Permiso concedido

    A->>S: OrdersService.create(dto)
    S->>P: Buscar cliente (ruta, tipo)
    P->>DB: SELECT client, route
    DB-->>P: Cliente con ruta y tipo
    S->>S: Calcular precios por línea
    Note over S: precio = base + (tarifa × factor)
    S->>P: Crear Order + OrderLines
    P->>DB: INSERT order, order_lines
    DB-->>P: Order creado
    S->>P: Crear OrderStatusLog (REGISTERED)
    P->>DB: INSERT status_log
    DB-->>P: Log creado
    P-->>S: Order completo
    S-->>A: OrderResponseDto
    A-->>N: 201 Created + order data
    N-->>U: Muestra pedido creado ✓

9. Flujo Técnico — Impacto en Inventario

sequenceDiagram
    participant U as Jefe Producción
    participant A as API
    participant PS as ProductionService
    participant IS as InventoryService
    participant DB as PostgreSQL

    U->>A: POST /api/production/:id/complete
    A->>PS: complete(productionId, resultDto)

    PS->>DB: Obtener orden de producción + inputs
    DB-->>PS: Orden con MP utilizada

    Note over PS: Registrar outputs reales

    PS->>IS: registrarSalidaMP(items, cantidades)
    IS->>DB: INSERT movement (PRODUCTION_OUT)
    IS->>DB: UPDATE inventory_item SET stock = stock - qty
    DB-->>IS: ✓ Stock MP actualizado

    PS->>IS: registrarEntradaPT(productos, cantidades)
    IS->>DB: INSERT movement (PRODUCTION_IN)
    IS->>DB: UPDATE inventory_item SET stock = stock + qty
    DB-->>IS: ✓ Stock PT actualizado

    Note over PS: Calcular métricas
    PS->>PS: rendimiento = output / input × 100
    PS->>PS: merma = input - output
    PS->>PS: costoReal = costoMP / outputQty

    PS->>DB: UPDATE production_order SET status = COMPLETED
    DB-->>PS: ✓

    PS->>IS: verificarAlertas()
    IS->>DB: SELECT items WHERE stock <= minStock
    DB-->>IS: Items con stock bajo

    PS-->>A: ProductionResultDto
    A-->>U: 200 OK + métricas de producción
